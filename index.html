Herramientas necesarias
Tres programas gratuitos que necesitás tener instalados antes de empezar
▾

Instalá estas tres herramientas antes de continuar. Hacé clic en cada enlace para ir a la página oficial de descarga:

  • Git Bash — git-scm.com/download/win
    Es la consola de comandos que vamos a usar durante toda la instalación. Seguí este video de configuración para asegurarte de que quede bien configurado.

¿Nada de esto requiere permisos de administrador? VSCode y Git Bash tienen opciones de instalación "solo para el usuario actual" que no piden contraseña. Python también funciona sin admin si elegís la opción de instalación por usuario.
Parte 1 Instalar Node.js sin permisos de administrador

Node.js es el motor que necesita Claude Code para funcionar. Como no podés usar el instalador normal, vamos a usar la versión portable: un ZIP que no requiere instalación.

1
Descargar Node.js en formato ZIP
Buscá el archivo .zip — NO el instalador .msi
▾
  1. Abrí tu navegador y andá a nodejs.org/en/download
  2. Buscá la sección de descargas para Windows
  3. Descargá el archivo con extensión .zip. El nombre va a ser algo como: 
    node-v22.x.x-win-x64.zip
⚠️
No descargues el .msi Si ves dos opciones para Windows, elegí siempre el .zip. El instalador .msi va a pedirte contraseña de administrador y no va a funcionar.
2
Extraer el ZIP en tu carpeta de usuario
Creá la estructura de carpetas y descomprimí el archivo
▾
  1. Abrí el Explorador de Windows
  2. Navegá a C:\Users\TU_USUARIO\ (reemplazá TU_USUARIO por tu nombre de usuario real)
  3. Creá una carpeta nueva llamada tools
  4. Dentro de tools, creá otra carpeta llamada nodejs
  5. Extraé todo el contenido del ZIP dentro de C:\Users\TU_USUARIO\tools\nodejs
✅
¿Cómo saber que lo hiciste bien? Dentro de la carpeta nodejs deberías ver el archivo node.exe. Si lo ves, vas bien.
3
Agregar Node.js al PATH de tu usuario
El PATH es la lista de rutas que Windows consulta cuando escribís un comando
▾
ℹ️
¿Qué es el PATH? Cuando escribís un comando como node en la consola, Windows busca en una lista de carpetas llamada PATH. Si Node.js no está ahí, el sistema no lo va a encontrar. Vamos a editarlo solo para tu cuenta — esto no requiere permisos de administrador. También podés seguir este video de referencia.
  1. Abrí el buscador de Windows (la lupa en la barra de tareas o presioná la tecla Windows)
  2. Escribí "variables de entorno" y seleccioná la opción "Editar las variables de entorno de esta cuenta"
    ⚠️
    Ojo — hay dos opciones parecidas Elegí la que dice "de esta cuenta", NO la que dice "del sistema". La opción del sistema requiere permisos de administrador.
  3. Se abre una ventana con dos secciones. Trabajá únicamente en la sección de arriba que dice "Variables de usuario". Nunca toques la sección de abajo ("Variables del sistema").
  4. Buscá la variable llamada Path en la sección de arriba y hacé doble clic sobre ella
  5. Hacé clic en "Nuevo" y escribí la ruta completa: 
    C:\Users\TU_USUARIO\tools\nodejs
  6. Hacé clic en Aceptar en todas las ventanas para guardar
✅
Verificación Cerrá PowerShell si lo tenías abierto, abrilo de nuevo y escribí: 
node --version
Si ves un número de versión como v22.x.x, el paso fue exitoso.
📋
Si querés aprovechar y configurar todas las variables de entorno de una sola vez, consultá el cuadro completo de variables y rutas al final de esta guía.
Parte 2 Configurar Git Bash

Git Bash es la consola que vamos a usar para instalar Claude. Necesita saber dónde encontrar Node.js.

4
Abrir Git Bash y configurar el PATH
Decile a Git Bash dónde encontrar Node.js
▾
  1. Buscá Git Bash en el menú inicio y abrilo
  2. Pegá este comando (reemplazá TU_USUARIO por tu usuario real antes de pegarlo):
    Comando 1 de 2 — ejecutar primero
    printf 'export PATH="/c/Users/TU_USUARIO/tools/nodejs:$PATH"\n' > ~/.bashrc
  3. Presioná Enter, esperá que termine, y recién entonces pegá este segundo:
    Comando 2 de 2 — ejecutar después
    source ~/.bashrc
  4. Verificá que funcionó: 
    node --version
npm --version
    Ambos deberían mostrar números de versión.
⚠️
Pegá los comandos de a uno En Git Bash, si pegás varios comandos a la vez, los puede interpretar como uno solo y fallar. Pegá uno, presioná Enter, esperá que termine, y recién entonces pegá el siguiente.
💡
Si source ~/.bashrc falla con "command not found": El archivo puede haber quedado con formato incorrecto. Volvé a ejecutar el primer comando con printf (no uses echo) y luego intentá el source de nuevo.
Parte 3 Instalar Claude Code
5
Instalar Claude Code con npm
Un solo comando — puede tardar varios minutos
▾

En Git Bash, ejecutá este comando:

npm install -g @anthropic-ai/claude-code
ℹ️
Esto puede tardar varios minutos dependiendo de tu conexión a internet. Verás texto desfilando en la pantalla — es normal. Dejalo terminar sin interrumpirlo.
6
Agregar Claude al PATH de tu usuario en Windows
Decile a Windows dónde encontrar el comando claude
▾
  1. Abrí el buscador de Windows y buscá "Editar las variables de entorno de esta cuenta" (igual que en el Paso 3)
  2. Hacé doble clic en la variable Path de la sección de arriba ("Variables de usuario")
  3. Hacé clic en "Nuevo" y agregá esta ruta: 
    C:\Users\TU_USUARIO\AppData\Roaming\npm
    Esta es la carpeta donde npm guarda los programas que instalás con npm install -g
  4. Hacé clic en "Nuevo" otra vez y agregá también: 
    C:\Users\TU_USUARIO\.claude\bin
    Si esta carpeta no existe todavía, agregala igual — se va a crear durante la instalación.
  5. Hacé clic en Aceptar en todas las ventanas
📋
Si querés aprovechar y configurar todas las variables de entorno de una sola vez, consultá el cuadro completo de variables y rutas al final de esta guía.
PS
Paso extra para Windows: configurar variables de entorno desde PowerShell
Alternativa al método manual — ejecutá estos comandos en PowerShell, de a uno
▾

Si tuviste problemas con el método manual o preferís hacerlo todo desde la terminal, podés usar estos comandos de PowerShell. Abrí PowerShell (no Git Bash) y pegá cada línea de a una.

⚠️
Ejecutá de a uno Copiá y pegá cada comando por separado. Esperá que cada uno termine antes de continuar con el siguiente.
1 — Instalar Git (si todavía no lo tenés)
winget install --id Git.Git -e --source winget
2 — Configurar la ruta de Git Bash para Claude (permanente)
setx CLAUDE_CODE_GIT_BASH_PATH "C:\Users\$ENV:USERNAME\AppData\Local\Programs\Git\bin"
3 — Configurar la ruta de Git Bash para Claude (sesión actual)
$env:CLAUDE_CODE_GIT_BASH_PATH = "C:\Users\$ENV:USERNAME\AppData\Local\Programs\Git\bin"
4 — Instalar Claude Code vía script oficial de Anthropic
irm https://claude.ai/install.ps1 | iex
5 — Agregar la carpeta .claude al PATH (permanente)
setx PATH "$env:PATH;C:\Users\$ENV:USERNAME\.claude"
6 — Agregar la carpeta .claude al PATH (sesión actual)
$env:PATH = "$env:PATH;C:\Users\$ENV:USERNAME\.claude"
7 — Agregar la carpeta .local\bin al PATH (permanente)
setx PATH "$env:PATH;C:\Users\$ENV:USERNAME\.local\bin"
8 — Agregar la carpeta .local\bin al PATH (sesión actual)
$env:PATH = "$env:PATH;C:\Users\$ENV:USERNAME\.local\bin"
9 — Agregar Python al PATH (permanente)
setx PATH "$env:PATH;C:\Users\$ENV:USERNAME\AppData\Local\Programs\Python\Python313;C:\Users\$ENV:USERNAME\AppData\Local\Programs\Python\Python313\Scripts"
10 — Agregar Python al PATH (sesión actual)
$env:PATH = "$env:PATH;C:\Users\$ENV:USERNAME\AppData\Local\Programs\Python\Python313;C:\Users\$ENV:USERNAME\AppData\Local\Programs\Python\Python313\Scripts"
⚠️
Ajustá el número de versión de Python Los comandos de arriba usan Python313 (Python 3.13). Si instalaste otra versión, reemplazá Python313 por la que corresponda (por ejemplo, Python312 para Python 3.12 o Python314 para Python 3.14). Para verificar tu versión, abrí la carpeta C:\Users\TU_USUARIO\AppData\Local\Programs\Python\ y fijate cómo se llama la subcarpeta.
✅
Cerrá y volvé a abrir la terminal Una vez ejecutados todos los comandos, cerrá PowerShell y volvé a abrirlo para que los cambios de PATH tomen efecto.
💡
¿Cuál es la diferencia entre los comandos "permanente" y "sesión actual"? Los comandos con setx guardan el cambio de forma permanente (para futuras sesiones). Los comandos con $env: lo aplican solo mientras la ventana de PowerShell está abierta. Ejecutar ambos garantiza que funcione tanto ahora como la próxima vez que abras la terminal.
📋
Si querés ver el listado completo de todas las variables y rutas que se configuran en este paso, consultá el cuadro resumen de variables de entorno al final de esta guía.
7
Agregar el PATH de npm en Git Bash también
Para que la consola encuentre el comando claude
▾

En Git Bash, ejecutá estos dos comandos (de a uno):

echo "export PATH=\"$(npm config get prefix)/bin:\$PATH\"" >> ~/.bashrc
source ~/.bashrc
⚠️
Recordá ejecutar los dos comandos de a uno, esperando que cada uno termine antes de ingresar el siguiente.
8
Verificar que todo funciona
Si ves la versión, la instalación fue exitosa
▾
  1. Cerrá Git Bash
  2. Volvé a abrirlo
  3. Escribí: 
    claude --version
🎉
¡Si ves un número de versión, la instalación fue exitosa! Ya podés pasar al último paso para iniciar sesión.
Parte 4 Usar Claude Code por primera vez
9
Iniciar sesión
La primera vez te va a pedir que te autentiques con tu cuenta de Anthropic
▾

En Git Bash, escribí simplemente:

claude

La primera vez que lo ejecutés, te va a pedir que inicies sesión con tu cuenta de Anthropic (la misma que usás en claude.ai). Seguí las instrucciones en pantalla.

✅
¡Listo! Una vez que iniciaste sesión, Claude Code está funcionando. Podés abrirlo desde Git Bash o desde la extensión de VSCode.

💡
¿Cómo agregar todas estas rutas? Podés hacerlo de dos formas: manualmente desde el menú de Windows (buscá "Editar las variables de entorno de esta cuenta" y editá Path para agregar cada ruta), o automáticamente ejecutando los comandos del Paso PS en PowerShell. Ambos métodos logran lo mismo. Para la variable CLAUDE_CODE_GIT_BASH_PATH, usá el botón "Nueva" en la ventana de variables de usuario para crearla.
